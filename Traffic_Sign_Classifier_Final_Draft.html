Step 1: Dataset Summary & Exploration¶

The pickled data is a dictionary with 4 key/value pairs:

  • 'features' is a 4D array containing raw pixel data of the traffic sign images, (num examples, width, height, channels).
  • 'labels' is a 1D array containing the label/class id of the traffic sign. The file signnames.csv contains id -> name mappings for each id.
  • 'sizes' is a list containing tuples, (width, height) representing the original width and height the image.
  • 'coords' is a list containing tuples, (x1, y1, x2, y2) representing coordinates of a bounding box around the sign in the image. THESE COORDINATES ASSUME THE ORIGINAL IMAGE. THE PICKLED DATA CONTAINS RESIZED VERSIONS (32 by 32) OF THESE IMAGES

Pre-process the Data Set (normalization, grayscale, etc.)¶

The most fundamental thing to do is mean normalization. This allows the optimization program to converge faster instead of being skewed in some direction. The image data should be normalized so that the data has mean zero and equal variance.

I tried converting to grayscale, or even converting to the YUV color space outlined in the above paper. However, I read that using an initial 3 1x1 filter allows the model to learn it's own best form of color space transform. Seemed pretty cool, so let's go with that! (Spoilers, it gives a huge jump in accuracy)

Data Augmentation¶

The bias in the data set may probably be due to the nature of the probability of encountering these datasets. We're more likely to see a "Stop" sign rather than a "Falling rocks" sign. So to get rid of this bias we can augment the data by generating new images. We do this by "jittering" our images. So basically taking the image and translating it, rotating it, or shearing it ever so slightly to give the Neural net a different perspective.

However, what I found was that it was destroying my validation set accuracy, even though my test set accuracy was higher. So I decided to forego the augmented data for better short term results. (Although I will rue this)

def augment_brightness(image):
    
    """ Randomly change the britness of the image
    """
    #transforming the image from RGB to HSC
    image1 = cv2.cvtColor(image,cv2.COLOR_RGB2HSV)
    random_bright = .25+np.random.uniform()
    #adding random brightness to the image
    image1[:,:,2] = image1[:,:,2]*random_bright
    #transforming the image back to RGB
    image1 = cv2.cvtCOLOR(image1,cv2.COLOR_HSV2RGB)
    return image1

def transform_image(img,ang_range, shear_range, trans_range):
    
    """Changes the image's angle, shear, brightness, and translation"""
    
    #Rotate the image
    
    angle_rot = np.random.uniform(ang_range) - ang_range/2
    row,col,channel = img.shape
    rotation = cv2.getRotationMatrix2D((col/2,row/2),angle_rot,1)
    
    #Translate the image 
    
    trans_x = trans_range*np.random.uniform()-trans_range/2
    trans_y = trans_range*np.random.uniform()-trans_range/2
    translation = np.float32([[1,0,trans_x],[0,1,trans_y]])
    
    # Shear
    pts1 = np.float32([[5,5],[20,5],[5,20]])

    pt1 = 5+shear_range*np.random.uniform()-shear_range/2
    pt2 = 20+shear_range*np.random.uniform()-shear_range/2

    pts2 = np.float32([[pt1,5],[pt2,pt1],[5,pt2]])

    shear_M = cv2.getAffineTransform(pts1,pts2)
    
    Shear the image 
    p1 = np.float32([[5,5],[20,5],[5,20]])
    p1 = 5+shear_range*np.random.uniform()-shear_range/2
    p2 = 20+shear_range*np.random.uniform()-shear_range/2
    p2 = np.float32([[p1,5],[p2,p1],[5,p2]])
        
    
    img = cv2.warpAffine(img,rotation, (col,row))
    img = cv2.warpAffine(img,translation,(col,row))
    img = cv2.warpAffine(img,shear_M,(col,row))
    
    #Brighten the image randomly 
    #img = augment_brightness(img)
    
    return img

Neural Network Architecture¶

My Neural Network Architecture heavily employs dropouts. Being a UofT Alumni myself, I have to follow in the footsteps of AlexNet and Geoffrey Hinton, haha! However, on a more serious note the redundancy provided by the method makes a huge difference in the final accuracy. It ensures that the model doesn't overfit and translates well to the test set.

I used 3 1x1 filters to learn the images best color space transform. Following which I used a series of 3x3 convolutions with filter sizes doubling at 16, 32, 64, and 128. Max pooling was strategically employed to bring the image down to a 5x5 spatial area from 32x32.

import tensorflow as tf
from tensorflow.contrib.layers import flatten

keep_prob = tf.placeholder(tf.float32)

def LeNet(x):    
    # Arguments used for tf.truncated_normal, randomly defines variables for the weights and biases for each layer
    mu = 0
    sigma = 0.1
    
    #First convolutional layer to using 3 1x1 filters for the system to learn color mapping
    weight = tf.Variable(tf.truncated_normal(shape=(1,1,3,3),mean=mu, stddev=sigma))
    bias = tf.Variable(tf.zeros(3))
    conv_layer= tf.nn.conv2d(x, weight,strides=[1,1,1,1], padding = 'VALID')
    conv_layer = tf.nn.bias_add(conv_layer, bias)
    conv_layer = tf.nn. relu(conv_layer)
    
    weight1 = tf.Variable(tf.truncated_normal(shape=(3,3,3,16),mean=mu, stddev=sigma))
    bias1 = tf.Variable(tf.zeros(16))
    #Layer 1: Convolutional. Input = 32x32x3. Output = 30x30x32.
    conv_layer= tf.nn.conv2d(conv_layer, weight1,strides=[1,1,1,1], padding = 'VALID')
    conv_layer = tf.nn.bias_add(conv_layer, bias1)
    #conv_layer = tf.nn.max_pool(conv_layer, ksize = [1,2,2,1], strides=[1,2,2,1], padding = "VALID")
    conv_layer = tf.nn.dropout(conv_layer, keep_prob)
                        
    weight2 = tf.Variable(tf.truncated_normal(shape=(3,3,16,32),mean=mu, stddev=sigma))
    bias2 = tf.Variable(tf.zeros(32))
    #Layer 2: Convolutional. Input = 30x30x32. Output = 28x28x64-->14x14x64 with max pooling
    conv_layer= tf.nn.conv2d(conv_layer, weight2,strides=[1,1,1,1], padding = 'VALID')
    conv_layer = tf.nn.bias_add(conv_layer, bias2)
    conv_layer = tf.nn.max_pool(conv_layer, ksize = [1,2,2,1], strides=[1,2,2,1], padding = "VALID")
    conv_layer = tf.nn.dropout(conv_layer, keep_prob)
                        
    weight3 = tf.Variable(tf.truncated_normal(shape=(3,3,32,64),mean=mu, stddev=sigma))
    bias3 = tf.Variable(tf.zeros(64))
    #Layer 3: Convolutional. Input = 14x14x32. Output = 12x12x64.
    conv_layer= tf.nn.conv2d(conv_layer, weight3,strides=[1,1,1,1], padding = 'VALID')
    conv_layer = tf.nn.bias_add(conv_layer, bias3)
    #conv_layer = tf.nn.max_pool(conv_layer, ksize = [1,2,2,1], strides=[1,2,2,1], padding = "VALID")
    #conv_layer = tf.nn.dropout(conv_layer, keep_prob)
    
    weight4 = tf.Variable(tf.truncated_normal(shape=(3,3,64,128),mean=mu, stddev=sigma))
    bias4 = tf.Variable(tf.zeros(128))
    #Layer 4: Convolutional. Input = 12x12x64. Output = 5x5x128.
    conv_layer= tf.nn.conv2d(conv_layer, weight4,strides=[1,1,1,1], padding = 'VALID')
    conv_layer = tf.nn.bias_add(conv_layer, bias4)
    conv_layer = tf.nn.max_pool(conv_layer, ksize = [1,2,2,1], strides=[1,2,2,1], padding = "VALID")
    conv_layer = tf.nn.dropout(conv_layer, keep_prob)
    
    
    #Flatten the convolutional layers so we can connect to traditional fully connected layers 
    fc0 = flatten(conv_layer)    
    # Layer 5: Fully Connected. Input = 400. Output = 120.
    weights_fc1 = tf.Variable(tf.truncated_normal(shape=(5*5*128,128),mean=mu,stddev=sigma))
    bias_fc1 = tf.Variable(tf.zeros(128))
    fc1 = tf.add(tf.matmul(fc0,weights_fc1),bias_fc1)
    fc1 = tf.nn.relu(fc1)
    fc1 = tf.nn.dropout(fc1,keep_prob)
    
    #Layer 6: Fully Connected. Input = 120. Output = 84.
    weights_fc2 = tf.Variable(tf.truncated_normal(shape=(128,84),mean=mu,stddev=sigma))
    bias_fc2 = tf.Variable(tf.zeros(84))
    fc2 = tf.add(tf.matmul(fc1,weights_fc2),bias_fc2)
    fc2=tf.nn.relu(fc2)
    fc2 = tf.nn.dropout(fc2,keep_prob)
    
    #Final Layer: Fully Connected. Input = 84. Output = 43 for our 43 classes
    weights_final = tf.Variable(tf.truncated_normal(shape=(84,43),mean=mu,stddev=sigma))
    bias_final = tf.Variable(tf.zeros(43))
    logits = tf.add(tf.matmul(fc2,weights_final),bias_final)
                                
    return logits

correct_prediction = tf.equal(tf.argmax(logits, 1), tf.argmax(one_hot_y, 1))
accuracy_operation = tf.reduce_mean(tf.cast(correct_prediction, tf.float32))
saver = tf.train.Saver()

def evaluate(X_data, y_data):
    num_examples = len(X_data)
    total_accuracy = 0
    sess = tf.get_default_session()
    for offset in range(0, num_examples, BATCH_SIZE):
        batch_x, batch_y = X_data[offset:offset+BATCH_SIZE], y_data[offset:offset+BATCH_SIZE]
        accuracy = sess.run(accuracy_operation, feed_dict={x: batch_x, y: batch_y, keep_prob: 1})
        total_accuracy += (accuracy * len(batch_x))
    return total_accuracy / num_examples

Step 4 (Optional): Visualize the Neural Network's State with Test Images¶

While neural networks can be a great learning device they are often referred to as a black box. We can understand what the weights of a neural network look like better by plotting their feature maps. After successfully training your neural network you can see what it's feature maps look like by plotting the output of the network's weight layers in response to a test stimuli image. From these plotted feature maps, it's possible to see what characteristics of an image the network finds interesting. For a sign, maybe the inner network feature maps react with high activation to the sign's boundary outline or to the contrast in the sign's painted symbol.

Provided below is the function code that allows you to get the visualization output of any tensorflow weight layer you want. The inputs to the function should be a stimuli image, one used during training or a new one you provided, and then the tensorflow variable name that represents the layer's state during the training process, for instance if you wanted to see what the LeNet lab's feature maps looked like for it's second convolutional layer you could enter conv2 as the tf_activation variable.

For an example of what feature map outputs look like, check out NVIDIA's results in their paper End-to-End Deep Learning for Self-Driving Cars in the section Visualization of internal CNN State. NVIDIA was able to show that their network's inner weights had high activations to road boundary lines by comparing feature maps from an image with a clear path to one without. Try experimenting with a similar test to show that your trained network's weights are looking for interesting features, whether it's looking at differences in feature maps from images with or without a sign, or even what feature maps look like in a trained network vs a completely untrained one on the same sign image.
